# 🧠 コンセプト整理

あなたのアプリは一言でいうと：

👉 **「色 × 用途 × プロジェクト」で管理できるカラーパレットツール**

既存のパレット管理と違うのは、

* 色単体ではなく「役割」で管理
* プロジェクト（フォルダ）ごとに色設計できる

ここがかなり強いです。

---

# 🧩 機能設計（具体化）

## ① フォルダ（プロジェクト）単位管理

* 作品ごとにフォルダ
* 例：

  * キャラA
  * 背景セット
  * UIデザイン案

👉 DB的には `projects` テーブル

---

## ② 色 + タグ管理（コア機能）

各色に対して：

* 色情報

  * RGB
  * HSV
  * HEX
* タグ（複数可）

  * 例：

    * 「肌ベース」
    * 「影」
    * 「ハイライト」
    * 「背景・遠景」

👉 ここがUXのキモ

---

## ③ タグのフォルダ内スコープ

重要ポイント👇

> タグはフォルダごとに分かれる

つまり：

* 「赤」でも用途が違う
* 同じタグ名でも別プロジェクトでは別物

👉 DB構造：

* tags は project_id に紐づく

---

## ④ 色のバリエーション管理

かなり便利になる拡張👇

* 「基準色」から派生

  * 明るさ違い
  * 彩度違い
* 例：

  * 赤（ベース）

    * 赤（影）
    * 赤（ハイライト）

👉 HSVが活きるポイント

---

## ⑤ 表示切替（ここは重要）

ユーザー的に欲しい👇

* RGB / HSV / HEX 切り替え
* スライダー操作
* カラーピッカー

👉 UI的には

* Figmaっぽい操作感が理想

---

# 🗄️ データ構造（簡易設計）

```plaintext
Project
- id
- name

Color
- id
- project_id
- r, g, b
- h, s, v
- hex
- base_color_id（派生元）

Tag
- id
- project_id
- name

ColorTag（中間テーブル）
- color_id
- tag_id
```
